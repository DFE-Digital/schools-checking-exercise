Multiple journeys

Design changes

Change: Questions are split into individual pages.

Reason: Aligns with the Design System - Question pages (design-system.service.gov.uk). answers.

Severity: High

Change: Yes and No radio questions are not inline and Yes is the first option.

Reason: Helps the user understand what is really needed from them.

Severity: Medium

Change: On the evidence page the text area for Additional comments has been removed.

Reason: If we don't need to collect information, we shouldn't. It seems mandatory on the form, so including it made sense.

Severity: Medium

Change: Summary comes before Evidence screen in every journey where evidence is mandatory.

Reason: The user may not have their evidence to hand when entering the system. This gives them the opportunity to submit the answers to the forms and come back to upload evidence later.

Severity: High

Change: Back buttons are kept on every question page.

Reason: Aligns with the Design System - Back link (design-system.service.gov.uk.

Severity: Medium

Change: Every journey has a summary page.

Reason: Aligns with the Design System - Check answers (design-system.service.gov.uk). Keeps the service consistent.

Severity: High

Pages that appear in every journey

What pages are included
  • Start page - Check your performance measures data (Key stage 4)
  • Your details
  • Your pupil details
  • No - Check your answers before submitting a request
  • Yes - Who do you want to remove?
  • Summary pages - Check your answers before submitting request or uploading evidence
  • Upload your evidence
  • Upload your evidence - One document uploaded
  • Requests - One in progress
  • Requests - One complete
  • Application complete
  • Header - Your details
  • Header - Your pupil details

Design changes

Change: 'Your details' now includes a section giving the user information if they need to change their details.

Reason: Currently 'Your details' seems like a reference page with no call-to-action. Users shouldn't step through this page if there is no action to take.

Severity: Low

Change: Changed table structure for 'Your pupil details'. Table has a title and Name and UPN have moved to become column headings.

Reason: Currently 'Student name' is repeated in every row, this change avoids repetition throughout the table. This takes up less space and is easier for screen readers.

Severity: Low

Change: Pupil UPN has been added as a column in 'Your pupil details'.

Reason: UPN is referenced throughout the journey, especially when pupils need merged. If the user needs to reference this page it is useful to give them all pupil details needed throughout the journey.

Severity: Medium

Change: Pupil details are in alphabetical order (by last name).

Reason: There should be structure to the names. It's an assumption that if students need merged the names would be similar, so this could make it easier to spot. Another assumption is that the users would know pupils by name first, before their UPN.

Severity: Medium

Change: Header now has links to 'Your details', 'Your pupil details',' 'DfE account' and 'Sign out'.

Reason: Gives the user the option to access reference pages at any point during their journey. They can also access their account that they've signed in with, and they are able to sign out.

Severity: Medium

Change: Changed the 'Create a request' page to a question, 'Do you want to request to remove a pupil?'

Reason: Has the same behaviour as the current form, however is more clear and provides a call-to-action.

Severity: High

Change: When the user does not want to make a request, the structure of the summary page has changed. The radio buttons are removed in favour of a summary table where the user can change that specific answer.

Reason: Currently the radio buttons don't have an impact. The change uses a standard Design System pattern - Check answers (design-system.service.gov.uk).

Severity: High

Change: 'Who do you want to remove?' pupil selection is now an accessible autocomplete - Accessible autocomplete - Alpha Gov (github.com).

Reason: Some schools will have a lot of pupils, this gives them a way to find pupils through their name or UPN while still maintaining accessibility.

Severity: High

Change: Evidence upload page uses the File upload component - File upload (design-system.service.gov.uk).

Reason: Keeps the service in line with the Design System.

Severity: Medium

Flow changes

Change: Users can access another version of the 'Your details' page from the header. This version does not have a section giving information to change their details.

Reason: This can be used as a reference point and is accessible at any point during their journey.

Severity: Low

Change: Users can access another version of the 'Your pupil details' page from the header. This version has less information.

Reason: This can be used as a reference point and is accessible at any point during their journey.

Severity: Medium

Change: Check your answers before submitting request or uploading evidence follow the Check answers pattern - Check answers (design-system.service.gov.uk). The user can change specific answers from these screens.

Reason: This can be used as a reference point and is accessible at any point during their journey.

Severity: High

Change: Evidence upload has been restructured. Text areas that are included on the pages have been removed. If the text area is required (e.g for Not on roll), this has been added as a question to the journey.

Reason: This keeps the one question a page structure. This means the user will have less chance of timing out when uploading evidence if there is a required answer. The text areas aren't always relevant to uploading files, so this can minimise users assuming they're connected.

Severity: High

Admitted from abroad with English not first language

Design changes

Change: 'What is their country of origin?' country is now an accessible autocomplete - Accessible autocomplete - Alpha Gov (github.com)

Reason: Selects have accessibility issues, this allows the user to type the specific country they want.

Severity: Medium

Change: On the form 'Date of arrival' is marked as optional, but the user can't continue their journey without entering a date. We have assumed this is mandatory and now exists as it's own question.

Reason: If we don't need to collect information, we shouldn't. It seems mandatory on the form, so including it made sense.

Severity: High

Change: On the evidence page the text area for Additional comments has been removed.

Reason: It's not clear what we would expect from this. Removing it simplifies the journey.

Severity: Low

Change: On the form evidence is marked as optional, however the guidance says this is mandatory. The guidance is linked through the evidence screen.

Reason: Following the guidance.

Severity: High

Flow changes

Change: After splitting the 3 questions into separate pages, these have also been reordered. We ask

  • 'When did they arrive?', then
  • 'When did they enrol at school college in England?', then
  • 'When did they enrol at your school or college?'

Reason: If these dates are different, this will mean they are entered in chronological order. This is more straightforward for the user and means they won't have to jump around in the timeline.

Severity: Medium

Recommendations

Applies to: 'What is their first language?' page

Recommendation: Provide a clearer list of more descriptive options and include how this can impact the outcome.

This could even be restructured to more straightforward questions depending on what is needed from this question. For example,

  • 'Do you know their first language? Yes or No'
  • Yes - 'What is their first language? Accessible autocomplete'
  • No - 'Is this believed to be English, or have they refused to disclose this? Yes or No'

This will not cover all scenarios as it's not clear how these answers impact the user and the outcome.

Reason to change: The options are not fully clear. 'Not known but believed English' and 'Not known but believed other than English' seem to put the responsibility on the user, with no knowledge about how this impacts the outcome.

Applies to: Flow of the questions

Recommendation: Change the structure of these questions, for example

  • 'When did they enrol at a school or college in England?', then
  • 'Is this the same date that they enrolled at your school or college? Yes or No'
  • Yes - 'When did they enrol at your school or college?'

This helps break the question style up, helping the user understand the differences in questions.

Reason to change: 'When did they enrol at a school or college in England?' and 'When did they enrol at your school or college?' are extremely similar questions that could cause confusion among users.

Completed KS4 studies this academic year in year 11 at another school or college

Design changes

Change: Evidence screen has been removed.

Reason: The guidance says that evidence for this journey is optional.

Severity: Medium

Recommendations

Applies to: 'Where did they complete their year 11 studies this academic year?' page

Recommendation: Use an accessible autocomplete based on the Select a school pattern. Select a school (design.education.gov.uk). This will give the user the option to find the URN by searching for a school name, college name or URN.

Reason to change: If the user does not know the URN of the school or college they have to leave the service to find it. There doesn't seem to be an option to enter the school or college name either, assuming that this would be more memorable for the user. This breaks up their journey and makes it inefficient and more effort for them.

Social care involvement - including police or prison

Design changes

Change: Split 'Has John Smith been involved in a recent situation that has required social care involvement that has had a severe and profound effect on their ability to access education and/or sit exams?' and 'Has John Smith had recent police involvement which has had a severe and profound effect on their ability to access education and/or sit exams?'

These have been split into

  • 'Have they recently been involved with social services?' and
  • 'Have they recently been involved with the police?' and
  • 'Has this had a severe and profound effect on their ability to access education or sit exams?'

Reason: The current questions are wordy and difficult to understand, splitting them gives the user more time to understand what is being asked of them. This also aligns with what is asked in the guidance.

Severity: Medium

Flow changes

! Warning If routing is not going to be implemented, follow this flow without it instead.

Change: 'Has this had a severe and profound effect on their ability to access education or sit exams?' is only asked if the user answers yes to either

  • 'Have they recently been involved with social services?', or
  • 'Have they recently been involved with the police?'

Reason: According to the guidance and current questions these are the only times we need to know if the pupil has been impacted in a severe way. This change means we can ask the users only relevant questions and not increase their cognitive load unnecessarily.

Implementation: Link to implemented flow.

Severity: Medium

Recommendations

Applies to: Flow of the questions

Recommendation: Change the flow so the user can answer the minimum amount of questions to get to their summary and evidence page.

This means that when the user answers Yes to questions that fulfil a removal criteria they skip the remaining questions. For example,

  • 'Have they recently been involved with social services? Yes or No'
  • Yes - 'Has this had a severe and profound effect on their ability to access education and sit exams? Yes or No''
  • Yes - 'Have they sat any year 11 exams? Yes or No'
  • Journey goes to Summary, then Evidence.

Reason to change: The user has to answer every question in the flow even if they fulfil the first case of the pupil being involved with the police and this having a severe and profound impact on them. This could cause stress and worry that the pupil does not fulfil the right criteria.

Implementation: Link to ideal flow.

Year group change

Design changes

Change: 'Is John Smith's year group being changed to lower than 11 or higher than 11? and 'Please provide the year group John Smith should be moved to?' have been combined into one question. We now ask 'Which year group have they changed to?' and list the years.

Reason: Both questions aren't needed so this makes the journey more straightforward. It lets us route the form sooner and gives clearer answers.

Severity: Medium

Behaviour to change or implement

Behaviour: On summary pages when the user changes an answer and then clicks continue, it returns to the summary page.

Reason: Aligns with the Design System - Check answers (design-system.service.gov.uk).

Behaviour: Request is only in the state 'In progress' if the user exits the upload evidence screen before submitting.

Reason: The journey doesn't save or have an option to exit throughout. This gives the user the opportunity to come back to upload their evidence later.

Behaviour: When a request is In progress, the user can continue to upload evidence or delete the request. When a request is complete, the user can only view or delete the request.

Reason: The user wouldn't need to continue a request that is complete.

Behaviour: Once a user has a request made, their name should not appear on the pupil name list and merged name list.

Reason: Means the user won't accidentally choose the wrong name. Appearing in the initial pupil name selection could confuse them that they haven't made their request yet.

Behaviour: Once a pupil is selected, their name should not appear in the merged pupil list.

Reason: Means the user won't accidentally choose the same pupil to merge with.

Behaviour: 'Cancel and return to requests' from Evidence screens takes you to 'Your requests' with the request in progress.

Reason: This gives the user the opportunity to come back to upload their evidence later.
